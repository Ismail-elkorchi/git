# ADR Template

## Title

The Title section MUST state one decision title sentence.

## Decision

The Decision section MUST state one or more decision obligations.

## Rationale

The Rationale section MUST state the technical reasoning for the decision.

## Consequences

The Consequences section MUST state expected effects and follow-up actions.

## Invariants

The Invariants section MUST cite one or more Invariant IDs.

## Gates

The Gates section MUST cite one or more gate commands.

## Operational Rubric R

The Operational Rubric R section MUST define reliability impact.
The Operational Rubric R section MUST define resilience impact.
The Operational Rubric R section MUST define reversibility impact.
The Operational Rubric R section MUST define resource impact.
The Operational Rubric R section MUST define risk impact.

## Structured Disagreement

The Structured Disagreement section MUST define the proposer position.
The Structured Disagreement section MUST define the challenger position.
The Structured Disagreement section MUST define disagreement points.
The Structured Disagreement section MUST define resolution criteria.

## Learning And Validation Loops

The Learning And Validation Loops section MUST define a premortem linkage.
The Learning And Validation Loops section MUST define a blameless postmortem linkage.
The Learning And Validation Loops section MUST define double-loop learning outputs.
The Learning And Validation Loops section MUST define PDCA updates.
The Learning And Validation Loops section MUST define OODA updates.

## Metrics And Evidence Controls

The Metrics And Evidence Controls section MUST define the primary metric.
The Metrics And Evidence Controls section MUST define the counter-metric.
The Metrics And Evidence Controls section MUST define the control group.
The Metrics And Evidence Controls section MUST define the holdout group.
The Metrics And Evidence Controls section MUST define Goodhart safeguards.
The Metrics And Evidence Controls section MUST define Hyrum compatibility risk status.
The Metrics And Evidence Controls section MUST define frame semantics impact status.
The Metrics And Evidence Controls section MUST define deterministic acceptance checks.

## Research Lenses

The Research Lenses section MUST cite one or more RL IDs.

## Lens To Invariant Mapping

The Lens To Invariant Mapping section MUST map each cited RL ID to one or more Invariant IDs.

## Evidence Mapping

The Evidence Mapping section MUST map each cited Invariant ID to one artifacts/gates path.

## Traceability Record

The Traceability Record section MUST contain one json code block.
The Traceability Record json block MUST define the key named lenses.
The Traceability Record json block MUST define the key named invariants.
The Traceability Record json block MUST define the key named mappings.
The Traceability Record json block MUST define the key named evidence.
The Traceability Record json block MUST map every cited lens ID to one or more cited Invariant IDs.
The Traceability Record json block MUST map every cited Invariant ID to one evidence artifact path.
